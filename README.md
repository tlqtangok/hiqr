## 功能特性

- 📦 **单个HTML文件** — 无需安装、无需服务器、发送端完全离线
- 🔄 **连续二维码流** — 文件分割成帧，自动循环播放的二维码动画
- 📱 **自动识别设备** — 桌面端自动进入发送模式，手机端自动进入接收模式
- 🔀 **从任意帧开始扫描** — 接收端可以从中途开始扫描，仍能重组完整文件
- ✅ **CRC32完整性校验** — 每次传输都经过校验，确保数据准确
- 🖼️ **即时预览** — 图片立即显示；文本文件在预览框中展示
- 📋 **复制或下载** — 收到的文本可复制到剪贴板；所有文件可保存
- ⏸️ **暂停/手动导航** — 暂停自动播放或逐帧浏览
- 🔍 **2倍放大** — 放大二维码显示，便于远距离扫描

## 工作原理

```
┌──────────────────────────────────┐            ┌──────────────────────────────────┐
│         电脑 / 桌面端            │            │         手机                      │
│                                  │  ───────►  │                                  │
│  文件 → Base64 → 500字节分片     │  二维码   │  摄像头 → jsQR → 重组文件        │
│  二维码流, 每1.2秒一帧          │            │  CRC32校验 → 预览/下载            │
└──────────────────────────────────┘            └──────────────────────────────────┘
```

1. **发送端（电脑）：** 文件被读取、编码为Base64，并按~500字节分片。每片被包装成JSON帧，包含文件名、总帧数、帧索引和CRC32校验码。帧被渲染为二维码（版本38，纠错级别L），每**1.2秒**刷新一次，连续循环。

2. **接收端（手机）：** 摄像头使用jsQR实时解码二维码。帧可以以任意顺序到达 — 应用追踪已接收的帧，待所有帧收齐后组装文件。CRC32校验确保数据完整后才提供预览或下载。

## 快速开始

1. 在**电脑**浏览器中打开 `hiqr.html` — 无需服务器，直接双击文件即可。
2. 在**手机**浏览器中打开 `hiqr.html`（通过本地分享、USB等方式）。
3. 按照下面的 [使用方法](#使用方法) 进行操作。

> 使用右上角的**切换模式**按钮可在任何设备上手动切换发送/接收模式。

## 使用方法

### 发送模式（电脑端）

页面在桌面浏览器上自动进入**发送模式**。

1. 点击**选择文件**，选择任意文件。文件名下方会显示预估的帧数。
2. 点击**生成二维码** — 二维码流立即开始播放。
3. 拿起手机摄像头对准屏幕开始扫描。

![发送模式](img/hiqr_send.png)

**传输过程中的控制按钮：**

| 按钮 | 功能 |
|------|------|
| ⏸️ 暂停 / ▶️ 继续 | 暂停或恢复自动播放 |
| ⬅️ 上一帧 | 手动退后一帧 |
| ➡️ 下一帧 | 手动前进一帧 |
| 🔍 放大2倍 / 缩小还原 | 放大二维码便于扫描 |

> 二维码流会自动循环 — 手机可以在下一轮循环中补齐漏掉的帧。

---

### 接收模式（手机端）

页面在手机浏览器上自动进入**接收模式**。

1. 点击**📷 启动摄像头扫描**。
2. 将摄像头对准桌面屏幕上的二维码。
3. 进度计数器显示已收到的帧数（例如 `12 / 35`）。
4. 所有帧收齐后，文件被校验并展示：
   - **图片**（jpg、png、gif、webp、svg 等）立即显示。
   - **文本文件**（txt、md、json、csv、js 等）在预览框显示，有**复制**按钮。
   - **所有文件**都可用**下载**按钮保存。

![接收模式](img/hiqr_recieve.jpg)

> 您可以从**任意帧**开始扫描 — 不一定要从第1帧开始。

## 技术细节

| 参数 | 值 |
|------|-----|
| 二维码版本 | 38 |
| 纠错级别 | L（单个二维码容量~2634字节） |
| 原始分片大小 | ~500字节 |
| JSON包装后帧大小 | ~700–850字节 |
| 帧间隔 | 1.2秒 |
| 完整性校验 | CRC32 |
| 二维码编码库 | qrcode.js（内嵌） |
| 二维码解码库 | jsQR（内嵌） |
| 外部依赖 | **无** — 完全自包含的单个HTML文件 |

## 致谢

hiqr 内嵌了以下优秀的开源项目（均以 MIT 许可证授权），无需任何外部网络即可运行：

| 库 | 用途 | 仓库 |
|----|------|------|
| [qrcode.js](https://github.com/davidshimjs/qrcodejs) by davidshimjs | QR 码生成（发送端编码） | github.com/davidshimjs/qrcodejs |
| [jsQR](https://github.com/cozmo/jsQR) by Cosmo Wolfe | QR 码解码（接收端摄像头扫描） | github.com/cozmo/jsQR |

此外，以下算法/规范的纯 JS 实现由本项目自行编写：

- **CRC32**（多项式 `0xEDB88320`，公有领域标准算法）— 用于传输完整性校验
- **inflateRawJS**（RFC 1951 deflate-raw）— 用于在 Safari 等不支持 `DecompressionStream('deflate-raw')` 的浏览器上作为 fallback 解压缩
